1. 项目目标

构建一套：

面向实体餐厅

支持手机扫码点餐

前后端分离、移动端优先

部署在 Vercel

使用 MongoDB Atlas 免费云数据库

的订餐系统，用于真实门店日常运营。

系统目标：

多桌并发稳定点餐

云端数据持久化（无丢单）

后厨快速出餐管理

2. 用户角色
角色	说明
顾客端用户	扫码后自助点餐
服务端用户	店员 / 后厨 / 管理员
3. 功能需求
3.1 顾客端功能
3.1.1 桌号输入页

输入桌号（必填，仅数字）

校验后进入菜单页面

3.1.2 菜单浏览页

菜品分类固定为：

海鲜 / 肉类 / 菜类 / 豆腐 / 汤类 / 饮料 / 其他

单个菜品信息：

名称

图片

价格

备注说明

操作：

加入购物车

数量增减

3.1.3 购物车

右上角常驻购物车按钮

支持：

修改数量

删除菜品

实时计算总价

3.1.4 提交订单

展示：

桌号

菜品列表

总价

提交后：

生成订单

状态为「未出餐」

提示线下付款

说明：系统不涉及任何线上支付逻辑。

3.2 服务端功能
3.2.1 管理入口

首页右上角隐藏入口

密码：88888888

3.2.2 订单管理

默认展示：未出餐订单

每条订单包含：

桌号

下单时间

菜品明细（名称 / 数量 / 单价）

总价

操作：

标记“已出餐”

查看历史订单

3.2.3 菜单管理

菜品：

新增

修改（价格 / 分类 / 图片 / 备注）

删除

3.2.4 数据清理

一键清空订单数据（不影响菜单）

二次确认防误操作

4. 非功能需求（Serverless 场景）

所有接口必须无状态

单接口响应 < 500ms

支持 ≥ 50 并发请求

数据库存储必须云端化
